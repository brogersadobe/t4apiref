template<typename _signal_traits >
void tetraphilia::imaging_model::TransparencyGroup< _signal_traits >::SetClipMap ( const signal_type *  map)
inline 
 Associates with the transparency group a map through which the result values of the clip step are transformed.

 TransparencyGroups in Tetraphilia are always "array-like," i.e., individual components are always byte-aligned
 (almost always one byte per component).  Contents of TransparencyGroups are frequently then sampled into surfaces,
 which may or may not be array-like.  Non-array-like surfaces frequently perform halftoning, which generates more
 visually pleasing results for large colored objects and images, but much worse results for bezier edges and text.

 The displeasing cases both correspond to results generated from the application of fractional shape:

 <code>
 Cr = ( 1 - fclip ) * Cb + fclip * Cs
 </code>

 where 0 < fclip < 1.

 The "clip map" adjusts this computation:

 <code>
 if ( fclip == 0 )
 Cr = Cb
 else if ( fclip == 1 )
 Cr = Cs
 else
 Cr = m_clipMap[ ( 1 - fclip ) * Cb + fclip * Cs ]
 </code>

 The expectation is that m_clipMap maps arbitrary values from the TransparencyGroup's domain to those values in that
 domain which will be unmodified in the dithering process.  Note that pure background and pure foreground values pass
 through unchanged, and thus may trigger halftoning.

 If a client is using Tetraphilia's halftoning services, clients can just use GetSurfaceClipMap to get the correct
 clip map.  For example:

 <code>
 typedef ByteSignalTraits<app_traits> signal_traits;
 typedef typename signal_traits::signal_type signal_type;

Define surface traits for a packed 4-bit gray surface template<typename _app_traits> struct FourBitGraySurfaceTraits { typedef _app_traits app_traits; static constexpr bool array_like = false;

typedef unsigned char block_type; enum { pixels_per_block = 2 }; enum { pad_bits = 0 }; enum { num_components = 1 }; enum { bits_per_pixel = 4 }; static constexpr bool little_endian = true;

typedef ComponentSize1<4> component_sizes; };

const signal_type* map = imaging_model::GetSurfaceClipMap( ArgType<signal_traits>(), ArgType<FourBitGraySurfaceTraits>() ); myGroup->SetClipMap( map );
