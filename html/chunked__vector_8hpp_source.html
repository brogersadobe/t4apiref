58 /*
59  Traits metafunction allowing a client to request a "stable" end iterator on the chunked_vector.
60 
61  Non-end iterators in chunked_vectors are always "stable," i.e., if you have an iterator pointing to
62  a valid element, and you don't erase it or any elements prior to it, or call clear(), or assign
63  to the chunked_vector, that iterator will remain valid even if you do other mutations (most notably,
64  push/emplace_back).
65 
66  A chunked_vector with a stable end iterator extends this property to the end() iterator. Specifically,
67  if you call end(), then call push_back, the result of that earlier call to end() will be a valid iterator
68  pointing to the new last element. This is a very useful property if you are using a chunked_vector as a
69  "sequence of sequences," where the end of one range is also the start of the next, _and_ you are building
70  the ranges in parallel with the elements of the chunked_vector.
71 
72  For example:
73 
74  struct my_traits;
75  template<> struct stable_end_iterator<my_traits> : public std::true_type {};
76 
77  chunked_vector<int, t4::allocator<int>, my_traits> vec;
78  auto it1 = vec.end();
79  vec.push_back(1); vec.push_back(2);
80  auto it2 = vec.end(); // it2 is now an end iterator, but we will safely dereference it later (after a push) because it is "stable"
81  vec.push_back(3); vec.push_back(4);
82  auto it3 = vec.end();
83 
84  std::cout << "first range: ";
85  std::for_each( it1, it2, []( int x ){ std::cout << x << ' '; } );
86  std::cout << "\nsecond range: ";
87  std::for_each( it2, it3, []( int x ){ std::cout << x << ' '; } );
88  std::cout << '\n';
89 
90  This program will write:
91 
92  first range: 1 2
93  second range: 3 4
94 
95  The above program's behavior is undefined if the stable_end_iterator trait is set to false_type.
96 
97  There is a cost to using this trait: the chunked_vector must at all times maintain storage for one more
98  element than it is actually using at all times. This means that the default constructor (for typical allocators)
99  allocates memory and therefore can throw.
100 
101  Default value: false
102 */
103 template<typename Traits>
104 struct stable_end_iterator : public std::false_type {};

122  std::vector workalike that stores objects in chunks, allowing for better
123  memory usage in exchange for slightly slower iteration.

127  storage is broken up in chunks).
