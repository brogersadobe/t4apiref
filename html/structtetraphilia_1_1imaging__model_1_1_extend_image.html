Detailed Description

template<typename _signal_traits>
struct tetraphilia::imaging_model::ExtendImage< _signal_traits >

Class for performing image fringe computation (i.e. image extension).

There are various methods for extending the edge of an image. These include 1) zero extension, 2) replication, and 3) reflection. The performance of these methods varies considerably and deemed not acceptable for our purposes.
The edge extension is a hard problem generally, since we are trying to extrapolate the data that we do not have. Speedo uses a fast line-fitting algorithm to find the slope of an imaginary line on the edges of the image. Speedo dynamically adjust the number of pixels that contribute to the slope computation based on the filter type. This slop is used to extrapolate the fringe pixels values (a fast operation).
For an MxN image Speedo needs to perform 2M+2N line fitting steps total.

Notes

ExtendImage class, only extends the "interesting planes" or "interesting subrange" of a pixel.

Bugs

1) Only ArrayLike AIMPixelBufferLayout are supported. 2) Only Planar AIMPixelBufferLayout are supported.

Extends the edge of an image.

Extends the data portion of storage2D that is specified by dataRect into the fringe area specified by fringeRect. These include 1) zero extension, 2) replication, and 3) reflection. The performance of these methods varies considerably and deemed not acceptable for our purposes. The edge extension is a hard problem generally, since we are trying to extrapolate the data that we do not have. Speedo uses a fast line-fitting algorithm to find the slope of an imaginary line on the edges of the image.
Speedo dynamically adjust the number of pixels that contribute to the slope computation based on the filter type. This slop is used to extrapolate the fringe pixels values (a fast operation).
For an MxN image Speedo needs to perform 2M+2N line fitting steps total.

Parameters:

storage2D describes input source image - the bits that that are used as source to compute the fringe pixels.

dataRect input source image bound - where within the src the actual aquaired signal exists.

storage2Dlayout image pixel layout - description of the pixel format of 'storage2D'; this must be any "array-like" layout (can either be chunky or planar).

fringeRect input image fringe bound - where the compute values will be written and is computed based on the values of the pixels near the source image's (dataRect) edge.

kind describes the edge extension method

valueToReplicate

tilingStepping if tiling this must be a nonzero pointer to real_point_type containing stepping information which is at least as large as the dimensions of dataRect.

Return Value:

None.

The current implementation of the ExtendImage assumes only arraylike pixel_buffer_type can have fringe. Hence, when storage2Dlayout->IsArrayLike() is not true, the dataRect is returned as fringe. We return the true fringeRect for any other case or when storage2Dlayout is null.

Parameters:

dataRect input source image bound - where within the src the actual acquired signal exists.

quality describes quality levels for bitmap image sampling.

origMeasureOfMatScaleFactor description of "starting measure of matrix scale factor" Speedo performs auto smoothing and auto sharpening when scaling up or down images. Speedo uses the "measure of scaling factor" to fully automate this process. The are times that an image might go through multiple transformations (i.e. we might break the "transform" into two matrix) and using different "quality". In these cases, to produce the best possible looking image the system needs to know the origMeasureOfMatScaleFactor that was computed from the "origTransform".

tileStepping nonzero pointer when tiling to real_point_type containing stepping information which is at least as large as the dimensions of dataRect.

storage2Dlayout an optional image pixel layout - description of the pixel format of 'storage2D'; this must be any "array-like" layout (can either be chunky or planar). storage2Dlayout is an optional parameter and can be null. If storage2Dlayout is non-null and is not an "array-like" layout; then the 'dataRect' is return as the "fringe rectangle".

Return Value:

The fringe bound for the given set of input.
