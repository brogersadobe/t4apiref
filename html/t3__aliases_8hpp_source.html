12 /*
13  Definition of namespace aliases inside the t4 namespace, to make accessing the cumbersomely-named T3
14  namespaces more convenient and readable.
15 
16  Most t4 aliases to t3 types are appended to the end of the files in which the t3 type is defined (e.g.:
17  the definition of t4::byte_signal_traits at the end of tetraphilia/imaging_model/signaltraits.hpp).
18  Two very important aliases are defined in this header file:
19 
20  The namespace "tetraphilia" is aliased to "t3"
21  The struct "tetraphilia::TetraphiliaAppTraits" is aliased to "t4::app_traits"
22 
23  This is a step towards phasing out app_traits (SLAGIATT); we populate the t4 namespace with much shorter
24  aliases for commonly used Tetraphilia types (i.e., types in the tetraphilia namespace). Most of this is
25  only possible now that there is now only one valid model of the "app_traits" concept (tetraphilia::TetraphiliaAppTraits).
26 
27  Here's a list of aliases, although these definitions are provided by the header files in which the "old name"
28  is defined.
29 
30  OLD NAMES NEW NAMES
