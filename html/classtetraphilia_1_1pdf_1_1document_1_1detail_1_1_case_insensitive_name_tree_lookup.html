Detailed Description

A 'search_traits' supporting a case insensitive search of a name tree. There is currently exactly one use case for this type of search: for PDF.Next, the Resources tree in the alternate parameter dictionary. Keys passed into these methods are presumed to be lower case and encoded as UTF8. Keys in the tree are presumed to be of type 'text string'.

This class does not implement is_in_range() or key_compare() and so is only suitable for algorithms that do not use the "Limits" array. It cannot, for for example, cannot be used with TreeFindFast. This makes sense because case-insensitive name tree lookup must be done with brute force, stepping through the entire name tree. And there is no way to implement a canonical lexical case-insensitive comparison, since sort-order is locale dependent.
