43 /*
44  Class to help compute minimum allocation size for a sequence of distinct
45  blocks of memory with different sizes and alignment requirements.
46 
47  An aligner is initialized with a "guaranteed block alignment," which is
48  the guaranteed minimum alignment of any pointer value returned by a hypothetical
49  allocator. For example, if you're planning to allocate a block of memory
50  using ::malloc, you would initialize the aligner with alignof(std::max_align_t)
51  (since malloc promises always to return a value aligned enough to store an
52  instance of std::max_align_t).
53 
54  Having initialized an aligner, you then give it a sequence of size/alignment
55  pairs, either using push() or operator()(). Once that is done, call aligner::get()
56  to retrieve the minimum number of bytes to request from your allocator such that
57  the resulting block is guaranteed to be able to fit (in order) a sequence of
58  blocks at least as large as the specified sizes, with each block aligned to
59  the specified alignment.
60 
61  Alignment values (including the guaranteed_block_alignment) must be powers of 2.
62  0 is not allowed.
63 
64  Example:
65 
66  Suppose you want to allocate a block of memory containing an instance of
67  control_block, followed by a string containing n chars, followed by the NEON type
68  int8x8x2_t from an allocator that promised to return 4-byte aligned values:
69 
70  t4::aligner a( 4 );
71  a.push<control_block>();
72  a.push( n, 1 );
73  a.push<int8x8x2_t>();
74 
75  void* p = my_allocator.alloc( a.get() );
76 
77  control_block* cb = t4::align_and_advance<control_block>( p );
78  char* str = t4::align_and_advance( n, 1, p );
79  int8x8x2_t* vec = t4::align_and_advance<int8x8x2_t>( p );
80 
81  Note that cb, str and vec are pointers to uninitialized raw memory. Practicing
82  RAII properly is very tricky in these cases; only do this if you know what you
83  are doing and--perhaps more importantly--you've established that this is worth
84  doing. make_shared uses a technique like this to handle a shared pointer's
85  control block and object memory with a single allocation.
86 */

186 /*
187  Given an alignment, size, pointer to a buffer and that buffer's size, return the next
188  aligned pointer, and advance p to the end of that pointer's region (i.e., result + sz).
189 
190  If there is insufficient space, or p is nullptr, returns nullptr and does not modify p
191  or size_remaining.
192 */
193 inline void* align_and_advance( size_t alignment, size_t sz, void*& p, size_t& size_remaining )
194 {
195  if ( p )
196  {
197  auto result = std::align( alignment, sz, p, size_remaining );
198  if ( result )
199  {
200  p = add_bytes( result, sz );
201  size_remaining -= sz;
202  }
203 
204  return result;
205  }
206 
207  return nullptr;
208 }

212  the buffer, and advance the pointer to the end of that region (i.e., result + sz). Behavior

214 */
