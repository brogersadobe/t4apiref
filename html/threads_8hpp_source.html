40 /*
41  Helper class which allows algorithms to efficiently yield to other cooperative (ContextLock-
42  holding) threads without the overhead of expensive TLS accesses.
43 
44  The whole point to CostBasedConditionYielder is to allow potentially expensive algorithms
45  to yield periodicially even if the algorithm's natural "frequency" is high. For example,
46  consider a bezier path reduction algorithm which contains a loop that iterates over every
47  scan line. If the algorithm yielded every single scan line, we'd yield at a far higher
48  frequency than necessary for responsiveness, and spend an unnecessarily large amount of
49  time context switching. But if we don't yield at all during the loop, the yield frequency
50  might be too low.
51 
52  So, CostBasedConditionalYielder allows us to amortize several natural algorithmic steps
53  across a single context switch. The mechanism is simple: increment an integer by a certain
54  "cost" value; if it exceeds a threshold constant then perform a "real" context switch.
55 
56  "Real" multithreading complicates this a little bit. As we gradually climb out from under T3's
57  original cooperative architecture, we need to carefully synchronize access to the cost counter,
58  which is logically a thread global. We have code that calls thread_yield() that is sometimes
59  executed while holding a context lock, and sometimes not (e.g., the bezier reducer).
60 
61  TLS accesses are actually quite expensive. But consider the two following mitigating facts:
62 
63  1) If a piece of code is not holding the ContextLock, it does not need to yield.
64  2) If a piece of code _does_ hold the ContextLock, it doesn't need to using expensive
65  synchronization primitives like std::atomic or thread_specific_ptr to manage access.
66 
67  The upshot of this is, once you're inside a single-threaded component of an algorithm (such
68  as bezier reduction), we can manage this update by testing for the existence of a ContextLock,
69  and conditionally updating accumulated cost by simple direct access to a global integer,
70  if and only if the ContextLock is held. The CostBasedConditionalYielder class encapsulates
71  this logic for a client.
72 
73  Note that in the longer term, our goal is to get rid of the ContextLock altogether, at which
74  point yielding itself is no longer a useful concept, so we don't need this code. But as
75  long as we have code that requires a ContextLock, this has to be the mechanism by which
76  cost-based yielding is managed.
77 */
78 class CostBasedConditionalYielder
79 {
80 public:
81  CostBasedConditionalYielder( void )
82  {
83  if ( IsContextLocked() )
84  m_costCounter = &thread_detail::TLSVars::g_costCounter;
85  else
86  m_costCounter = nullptr;
87  }
88 
89  /*
90  Yield to another thread conditionally based on an accumulated computational weight-counter.
91 
92  This is provided so that an algorithm that performs a high-frequency operation on data sets that
93  vary greatly in size can periodically yield without incurring a severe performance penalty.
94  "Cost" is added to an internally managed counter; if that counter exceeds an internally defined
95  limit (currently 1000000), it actually releases the context lock with the intention of yielding
96  control to another thread, if any.
97 
98  When assessing the cost of a single iteration of an algorithm's operation, the idea is to select
99  a cost small enough such that the number of iterations required to add up to a million is comprised
100  of a number of instructions that dwarfs the cost of an actual context switch, but isn't so large
101  that the algorithm can starve the context during a particularly long operation.
102 
103  In practice, however, you're better off just determining an algorithm's cost experimentally. Just
104  choose a value as large as you can without perceptibly affecting the performance of the algorithm.
105  */
106  void Yield( int cost )
107  {
108  if ( m_costCounter )
109  {
110  *m_costCounter -= cost;
111  if ( *m_costCounter <= 0 )
112  ThreadYield();
113  }
114  }
115 
116 private:
117  int* m_costCounter;
118 };
