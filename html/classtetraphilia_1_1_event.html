Construct an Event object.

This call constructs an Event in the given application context with initial state determined from the initiallySignaled parameter. The manualReset parameter determines whether or not the event object must be manually reset after being signaled. In practice, if manualReset is true then all waiting threads will always be resumed when the object is signaled. If manualReset is false, then only one thread will be resumed before the event is automatically reset to be not signaled.

void tetraphilia::Event::Signal ( void  )

Set the event object to be in the SIGNALED state.

If the object is defined with manualReset = true then all waiting threads will be resumed. If the object is defined with manualReset = false then only one waiting thread will be resumed, after which the event object's state will be reset automatically to be not signaled.
