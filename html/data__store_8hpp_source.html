48 /*
49  Given a model of "data store device," create a data_store.
50 
51  A data store device must provide the following services:
52 
53  data_store_size get_size( Device&& d )
54 
55  Optional. Returns the size of the underlying device in bytes. If this is
56  not provided, queries will result in enumeration of all blocks in the device
57  until EOF is detected, so if this is not provided querying the length can be
58  very expensive.
59 
60  data_block get_block( Device const& d, data_store_position const& pos )
61 
62  Returns a data_block with a length of at least 1 representing the bytes at the
63  specified offset, unless pos >= get_size( d ), in which case a zero-length block
64  is returned.
65 
66  bool can_prefetch( Device&& d )
67 
68  Should return true if the underlying device does something useful with prefetch_bytes
69  calls. This is simply advisory; clients may call prefetch_bytes either way.
70 
71  void prefetch_bytes( Device const& d, data_store_byte_range const& range )
72 
73  Indicates to the underlying device that the specified byte range is likely to
74  be requested soon. Performance will be better if these bytes are fetched now, instead
75  of waiting for an explicit request.
76 
77  void reset_size( Device&& d )
78 
79  Reset the device's underlying notion of its size. This is deprecated and will be
80  rendered unnecessary (and removed) in a subsequent release.
81 
82  Device must be MoveConstructible. The above calls will happen on multiple threads without
83  explicit synchronization. It is expected that implementations of Device will use
84  mutex and a shared_ptr to mutable state to provide thread safety.
85 
86  It is required that Device return the same bytes for the same queries for the lifetime
87  of its enclosing data_store (with the exception of calls across the soon-to-be-deprecated
88  reset_size). If this contract cannot be fulfilled an exception should be thrown.
89 */
90 template<typename Device>
91 data_store make_data_store( Device&& ds );
