29 /*
30  About type conversion in t4
31 
32  The t4::convert function is a general value conversion function. Its intended use is for
33  conversion of regular types to similar regular types, perhaps with slightly different
34  representation, and to provide a facility for identifying when a conversion is possible and
35  when it is not.
36 
37  t4::convert has an extensible model--implementers may provide specializations of the
38  convert_impl template--but there is a philosophy behind t4::convert that it is important to
39  support. This philosophy is not necessarily the right foundation for generalized type conversion,
40  but it aligns well with T4's needs, particular as regards numerics and geometrical types.
41 
42  The basic idea is this: if a "from" type represents values from some abstract value space
43  (frequently with some sort of limits to the representable range--e.g., std::int32_t can only
44  represent integers between -2^31 and 2^31-1), and a "to" type represents values from a different
45  abstract value space with significant, but not complete, overlap. Some good examples of this:
46 
47  arbitrary integral type -> other arbitrary integral type
48  real-valued rectangle -> other real-valued rectangle
49  real-valued point -> other real-valued point
50 
51  We also support:
52 
53  arbitrary integral type -> floating-point type
54 
55  We do _not_ support the reverse (floating-point type -> integral type) as it's almost never
56  what you want--to be consistent with the philosophy stated above, we would expect that conversion
57  to fail for any floating-point value that is nonintegral. This is so rarely what a user actually
58  wants that we don't even support it and generate a compiler error. Instead, users wishing to convert
59  from a floating-point value to an integer value should use t4::floor, t4::round or t4::ceil, depending
60  on their requirements.
61 
62  t4::convert should really only be used for algebraic conversions between types that, at some level
63  or another, are modeling similar concepts; please don't use it for conversion to strings or stuff
64  like that.
65 
66  t4::convert is a template taking two parameters, one explicit specifying the target type, and one
67  that should be deduced from the single parameter. An example invocation might look like this:
68 
69  auto i = t4::convert<int>( some_size_t );
70 
71  This conversion will fail (and throw a std::overflow_exception) if some_size_t has a value that
72  can't be represented in an int.
73 
74  If you don't want an exception on failure to convert, requested a boost::optional type:
75 
76  auto i = t4::convert<boost::optional<int>>( some_size_t );
77 
78  Here, the return value will be a boost::optional<int>. If some_size_t has a value that can't be
79  represented in an int, an uninitialized optional is returned. Otherwise an optional with the mathematically
80  correct converted value is returned.
81 
82  About failure in conversion
83 
84  Conversions can fail. Typically conversions fail because the abstract value in to "from" type
85  is not representable in the "to" type (e.g., t4::convert<char>( 1000 )). For these cases t4::convert
86  will follow the rules above and either throw or return an uninitialized optional, depending on
87  the "to" type.
88 
89  In the case where an optional is requested, an exception _may_ still be thrown. In the course of
90  determining whether or not the conversion is possible exceptional conditions are permitted to arise.
91  And of course if the class itself consumes resources, its contruction may fail as well. The uninitialized
92  optional case should only be used where the "from" value being converted simply does not exist in the
93  representation space of the "to" type. All other sorts of failure should be signaled via a thrown
94  exception.
95 */
96 
97 /*
98  Implementers specialize this to provide conversion support across types
99 
100  Specializations of convert_impl must provide an operator() which does something
101  like this:
102 
103  namespace t4 {
104 
105  struct convert_impl<my_to_class, my_from_class>
106  {
107  // Implementations must return a boost::optional. This is used to indicate whether
108  // or not the conversion succeeded.
109  boost::optional<my_to_class> operator()( my_from_class const& from ) const
110  {
111  if ( can_create_my_to_class_from( from ) )
112  return create_my_to_class_from( from );
113  return {};
114  }
115 
116  // Optional: if a fail() method is provided, it must be [[noreturn]] and throw
117  // an appropriate exception. If this is not provided, t4::corrupt_data is thrown.
118  [[noreturn]] void fail( void ) const
119  {
120  throw cant_make_from_class_exception();
121  }
122  };
123 
124  } // end namespace t4
125 */
126 template<typename To, typename From, typename = void>
127 struct convert_impl;

129 /*
130  Convert the value to the specified result type if possible, or return an unitialized
131  optional.
132 
133  @param x The value to be converted to ResultType
134  @return If the conversion succeeds, the correct result inside an optional. If it fails,
135  an uninitialized optional is returned.
136 
137  Invocations will typically take the form:
138 
139  from_type x = get_from_type();
140  boost::optional<to_type> z = convert<boost::optional<to_type>>( x );
141 
142  The first template parameter identifies the desired result type. If
143  the conversion cannot be performed because the input value has a value
144  that can't be represented in the target space, an uninitialized optional
145  is returned..
146 
147  A convenience alias for the above is provided. The above is identical to:
148 
149  boost::optional<to_type> z = convert_optional<to_type>( x );
150 */
151 template<typename To, typename From>
152 constexpr std::enable_if_t<is_instantiation_of<To, boost::optional>::value,
153 To> convert( From&& x );
154 
155 // Convenience wrapper function for convert<boost::optional<To>>
156 template<typename To, typename From>
157 constexpr boost::optional<To> convert_optional( From&& x );
158 
159 /*
160  Convert the value to the requested type or throw an exception.
161 
162  @param x The value to be converted to the "To" type
163  @return If the value can be converted to To, the correct result. Otherwise failure is
164  signaled by an exception as dictated by the invoked implementation of convert_impl.
165 
166  Invocations will typically take the form:
167 
168  to_type z = convert<to_type>( x );
169 
170  The first template parameter identifies the desired result type.
171 */
172 template<typename To, typename From>
173 constexpr std::enable_if_t<!is_instantiation_of<To, boost::optional>::value,
174 To> convert( From&& x );
