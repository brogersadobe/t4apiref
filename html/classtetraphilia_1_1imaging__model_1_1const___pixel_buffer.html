Detailed Description

template<typename _signal_traits>
class tetraphilia::imaging_model::const_PixelBuffer< _signal_traits >

Object that describes a two-dimensional buffer of pixel data.

The PixelBuffer class represents a rectangular buffer of pixels. Each pixel is a set of pixel values; each pixel is an identically laid-out set of signal values. The format of the pixel buffer is given by the signal_type template parameter and the PixelBufferLayout data structure pointed to by the m_layout member.

PixelBuffers contain a pointer to a bounding rectangle; outside of this bounding rectangle the pixel values are undefined.

Because pixel buffers can be used to store anything from color to alpha to interleaved data, and perhaps other things I haven't yet thought of, we use the term "signal" to describe the values stored in each pixel. Most of the time these will either be color component values, or alpha coverage values.

The type of the signals contained inside the pixels of the pixel buffer is identified by the _signal_type template parameter. Read-only pixels are identified by a const-qualified signal type. Buffers of pixels where the signal type is undefined or unknown are identified by using a _signal_type of void or const void.

Note that all of the member variables are pointers to data structures over which the PixelBuffer instance asserts no ownership. This insures that pixel buffers, being only 12 bytes, can be copied very inexpensively; it is not recommended to store references or pointers to pixel buffers. Typically these pointers point to data owned by a PixelStore instance (which see).

See the documentation for the individual member variables for details.
