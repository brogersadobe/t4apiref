◆ GetTrueTypeEncodingTable()

template<typename app_traits >
virtual const std::uint16_t* tetraphilia::fonts::Font< app_traits >::GetTrueTypeEncodingTable ( NamedEncoding  namedEncoding,
bool  isSymbolic,
const char *const **  glyphNames = nullptr 
) const
pure virtual

returns a 256-long table that maps bytes in a string to glyphIDs for this font – only valid if this font uses kTrueTypeEncoding

"glyphNames" is an out parameter which, in the event that the encoding requested by namedEncoding is unavailable, returns a table mapping glyph IDs to glyph names. The caller can then use this table to map char codes in the requested encoding to the named glyph IDs. If the returned value is non-null then an encoding mismatch occurred and the implementation was able to provide the names.

The returned parameter is allocated in the current transient heap, and will only last as long as the current snapshot.
