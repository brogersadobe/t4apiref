32 /*
33  Storing C++ state in Java objects
34 
35  C++ objects are stored in Java objects as longs, which are large enough to be convertible to raw
36  pointers on both 32-bit and 64-bit platforms. These "longs" are called "proxies" here. The actual
37  pointer is found with a simple reinterpret_cast.
38 */
39 
40 /*
41  Given a jobject that extends com.adobe.t4.NativeProxy, return a pointer to
42  the C++ object contained therein. Will not throw an exception.
43 */
44 template<typename NativeType>

53 /*
54  Lifetime management of proxies is achieved by extending the "com.adobe.t4.NativeProxy" class.
55  This function will:
56 
57  1) Construct an instance of T on the heap passing args to the ctor
58  2) Attach the pointer to the NativeProxy instance
59 
60  At this point the created object (which can be retrieved using get_native_object()) is now
61  owned by the NativeProxy object. If an exception occurs at any point during this process
62  the created object is freed before the exception is propagated, so there is no possibility
63  of a leak.
64 */

66 void create_and_set_java_proxy( JNIEnv* env, jobject ths, TArgs&&... args );
