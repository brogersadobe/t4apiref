Detailed Description

Represents text for a mcid_content_element.

The text is stored in two forms.

The "primary" text is typically text used for display. For example, when text is extracted for FTPDF->HTML translation, the primary text appear in the HTML text nodes.

In some circumstances there is an "alternate" text representation. This is typically used, for example, when the primary text will be used with a converted font that has non-standard Unicode mappings. Such fonts typically make use of Unicode Private Use Area characters to select special glyphs in the converted font. In such cases the "alternate" text contains equivalent text using standard Unicode characters. It is suitable for purposes such as copy and search, and also for display using fonts other than those generated during translation.

template<typename T >
void t4::pdf::text_show::walk_text ( T &&  f) const
inline

Walks all the text in the text_show.

Internally text is stored as a sequence of chunks. This method iterates through these chunks and invokes a callback function on each. For chunks where the alternate text differs from the primary text, both the primary and alternate text are passed to the callback. In cases where the alternate text matches the primary text, only the primary text is passed.

Typically callers will see a sequence of chunks that alternate between having and not having alternate text.

Parameters
fText enumeration callback. Has the signature: void (const vector<uint32_t>&, optional<vector<uint32_t>>); where: the first parameter is the primary text the second parameter is the optional alternate text, which will be none if the alternate text matches the primary
