_CONCEPT_PDFDislayTraits Interface Reference

#include <displaytraits.hpp>

Detailed Description

/brief Traits concept whose models describe the feature set that should be enabled when rendering PDF.

The PDFDisplayTraits concept contains a set of enumerated constants describing features that the client can turn on and off. Typically, turning off a feature results in slightly smaller code, and slightly faster rendering, and slightly lower quality.

Example:


typedef MyPDFDisplayTraits
{
    ... continued below ...

kSupportNonIntegralAlpha

If this constant is set to 1 the /ca and /CA keys are respected in /ExtGState dictionaries. Files that use nonintegral values for these keys will render the corresponding objects transparently. If this constant is set to 1 the value of /ca and /CA keys are ignored, and all objects render opaquely.

Example:


    enum { kSupportNonIntegralAlpha = 1 };

kSupportNonNormalBlendModes – NOT SUPPORTED

This constant controls whether or not PDF blend modes (except for "Normal") are supported. If this is set to 1 the value of the /BM key in /ExtGState dictionaries is respected.

This feature is not yet supported. Clients should not enable it.


    enum { kSupportNonNormalBlendModes = 0 };

kSupportTransparencyGroups – NOT SUPPORTED

This constant controls whether or not PDF Transparency Groups are supported.

This feature is not yet supported. Clients should not enable it.


    enum { kSupportTransparencyGroups = 0 };

kSupportColorKeyMasks

This constant controls whether or not chroma key masking is supported. If it is off the chroma key mask is ignored.


    enum { kSupportColorKeyMasks = 1 };

kSupportSoftMasks – NOT SUPPORTED

This constant controls whether or not soft masks are supported. If it is off the soft mask keys in images and transparency groups are ignored.

This feature is not yet supported. Clients should not enable it.


    enum { kSupportSoftMasks = 0 };

kSupportSmoothShadeLevel

This constant controls the degree of support for smooth shades provded by Tetraphilia. There are three legal values:

  • 0: Indicates no smooth shades should ever be rendered.
  • 1: Indicates that only axial and radial smooth shades should be supported.
  • 2: Indicates that all 7 smooth shade types in PDF 1.7 are supported.

Axial and radial smooth shades are relatively common. The other forms of smooth shades are so incredibly rare the client may wish to decrease code size a little and only support axial and radial smooth shades.


    enum { kSupportSoftMasks = 0 };

kSupportType4Functions

This constant controls whether or not the Type 4 function parser is included in the build. Type 4 functions appear occasionally in some color space objects. If this is disabled objects rendered in that color space will either not appear or appear with potentially very wrong colors.


    enum { kSupportType4Functions = 1 };
