92 // 10 Jun 00:
93 // An additional optional template parameter was added to most of
94 // operator templates to support the base class chaining technique (see
95 // documentation for the details). Unfortunately, a straightforward
96 // implementation of this change would have broken compatibility with the
97 // previous version of the library by making it impossible to use the same
98 // template name (e.g. 'addable') for both the 1- and 2-argument versions of
99 // an operator template. This implementation solves the backward-compatibility
100 // issue at the cost of some simplicity.
101 //
102 // One of the complications is an existence of special auxiliary class template
103 // 'is_chained_base<>' (see 'operators_detail' namespace below), which is used
104 // to determine whether its template parameter is a library's operator template
105 // or not. You have to specialize 'is_chained_base<>' for each new
106 // operator template you add to the library.
107 //
108 // However, most of the non-trivial implementation details are hidden behind
109 // several local macros defined below, and as soon as you understand them,
110 // you understand the whole library implementation.

714 // is_chained_base<> - a traits class used to distinguish whether an operator
715 // template argument is being used for base class chaining, or is specifying a
