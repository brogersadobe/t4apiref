Detailed Description

Describes the layout of a pixel buffer.

The PixelBufferLayout data structure describes the geometry of a two-dimensional buffer of multi-component pixels.

const size_t tetraphilia::imaging_model::PixelBufferLayout::kReplicatedSignal = ~static_cast<size_t>(0)
static

Special constant value for replicated signals.

Set the m_numSignals field to this value when using a replicated signal (i.e., m_signalStride == 0). This is frequently used in buffers storing alpha values, where a pixel's alpha value is the same for each component.

◆ m_padSignalsAreUnused

bool tetraphilia::imaging_model::PixelBufferLayout::m_padSignalsAreUnused

Special tag indicating whether operations writing into the pixel buffer may overwrite unused bytes.

When associated with a writable pixel buffer, indicates whether or not the unused bytes are ignored by all other users of the pixel buffer. If this value is true it will frequently enable the use of faster pixel-processing algorithms.

ptrdiff_t tetraphilia::imaging_model::PixelBufferLayout::m_signalStride

The distance in bytes between adjacent signals within a single pixel.

The distance in bytes from the nth signal of the pixel at (x,y) to the (n+1)th signal of the pixel at (x,y). This can be negative for little-endian-style layouts.
