Detailed Description

template<typename search_client_type>
class tetraphilia::pdf::textextract::AllOccurrenceSearcher< search_client_type >

Object that manages a search with a fixed set of parameters within a PDF document.

An instance of Searcher searches a the document for a string of unicode characters matching a provided string under a set of search parameters (e.g., match case, whole word).

The Searcher template takes one template argument, the search_client_type. This type must model the _CONCEPT_search_client concept. The Searcher will periodically call various callbacks in the client to report found strings and fatal errors. It will also ask if pages are authorized before attempting to search them, and whether or not the search should wrap around when the end of the document is hit.

An instance of Searcher creates a worker thread which performs the actual searching of the document. When the worker thread locates an occurrence of the requested string in the document, it calls the client's HandleSearchFound() method and then waits on an Event object. This Event object is signalled when the client calls Searcher::Go(), presumably from the main thread when the user clicks the "find again" button.

template<typename search_client_type >
void tetraphilia::pdf::textextract::AllOccurrenceSearcher< search_client_type >::Go

Start or continue the search.

The Searcher maintains a worker thread which actually searches the document. This worker thread is constructed the first time the client calls Go(), and waits on an event whenever it reports a found string. Subsequent calls to Go() signal this event, causing the worker thread to search for the next match.

It is expected that the client will call Go() whenever the user clicks "find" or "find again."
