Detailed Description

template<typename tree_traits>
class tetraphilia::pdf::document::detail::tree_pos< tree_traits >

Represents a position in a name/number tree.

The position is represented by pairs of array and item_index values.

Current position in any Kids arrays is represented by a stack of array/index pairs. The array holds the contents for the current Kids array; each elelemet is an array. The index holds the position within that array. When traversal descends into a node with a Kids array, a new array/index pair is pushed onto the stack. The index value is initially invalid, but as the array is traversed the index in this topmost pair is incremented.

When traversal enters a Names/Nums array, the array and current index is similarly stored. The array holds the contents of the Names/Nums array; the elements are alternating key/value pairs. Here too the index holds the position within that array.

Logically this class can be thought of as a single stack including both Kids and Names/Nums arrays. From an implementation standpoint it proved more convenient to keep them separate. They are stored in different data members and have different APIs for manipulating.

(An alternative implementation was considered and even briefly used, in which pairs of iterators were stored instead of array/index pairs. One holds the current array position and the other holdsend(). This is workable, but was rejected because there is no way to modify the array from its iterator.)
