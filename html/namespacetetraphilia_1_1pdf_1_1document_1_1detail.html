◆ TreeFindFast()

template<typename search_traits >
boost::optional<tree_pos<typename search_traits::tree_traits> > tetraphilia::pdf::document::detail::TreeFindFast ( const t4::pdf::dictionary &  tree,
const typename search_traits::tree_traits::param_key_type &  key 
)
inline

Given a key, finds and returns an item in a name tree or number tree, or boost::none if the key doesn't exist.

Uses the "Limits" arrays to optimize the search. Name and number trees are optimized for key-based lookup, and by using the "Limits" dictionary the algorithm can prune out entire sub-trees. This search is fast, but because it doesn't visit every "Names" array it cannot also return the item's index (unlike TreeFindSlow).

Because it checks keys against the Limits array this method cannot be used for case-insensitive searching.

A valid way to check for existence of a key is to call this method and check for boost::none.

Parameters
treeThe name tree or number tree to search.
keyThe key to search for.
Returns
The pdf object found, or boost::none if the key doesn't exist.

◆ TreeFindSlow()

template<typename search_traits >
boost::optional<std::pair<tree_pos<typename search_traits::tree_traits>,size_t> > tetraphilia::pdf::document::detail::TreeFindSlow ( const t4::pdf::dictionary &  tree,
const typename search_traits::tree_traits::param_key_type &  key 
)
inline

Given a key, returns the position of the value within a name or number tree. The position is returned both as a tree_pos instance and as an absolute index within the tree.

The default implementation visits each Names/Nums array, but only examines individual entries if the Limits array indicates that the searched-for item resides there. This method is slower than TreeFindFast, but not drastically so. Because it checks keys against the Limits array this method cannot be used for case-insensitive searching.

It is valid to check for existence of a key by calling this method and checking for boost::none. However, this is less efficient than calling TreeFindFast() for the same purpose.

Parameters
treeThe name tree or number tree to search.
keyThe key to search for.
Returns
The tree_pos and index associated with the specified key, or boost::none if the key doesn't exist.
