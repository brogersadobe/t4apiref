69 /*
70  Instances of RasterPainter are only allocated out of the transient_heap, which knows the fully
71  derived type at the time of creation. As such they do not need a virtual dtor. Adding a virtual
72  dtor causes is_trivially_destructible be false, which compels the transient_heap to do a bunch
73  of additional bookkeeping on the object, which we'd like to avoid.
74 
75  This is also true for bezier samplers, image pipes, pixel producers, and several other classes
76  used to support rasterization. These objects are all allocated out of the current thread's
77  transient_heap, and deleted when the snapshot expires.
78 
79  In general, this warning is very valuable and helps us avoid leaks, so we want it enabled by
80  default. But this is an important case where we want to disregard it.
81 */
82 #ifdef __GNUC__
83 #pragma GCC diagnostic push
84 #pragma GCC diagnostic ignored "-Wnon-virtual-dtor"
85 #endif
