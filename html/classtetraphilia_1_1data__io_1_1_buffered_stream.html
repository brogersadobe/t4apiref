◆ Peek()

char tetraphilia::data_io::BufferedStream::Peek ( void  )
inline

Peek at the next character in the stream. Note that you must have called AtEOF sometime prior to this call and after the last call to Advance (if you have made such a call). In DEBUG builds this routine asserts this condition. In release builds it does not. In a release build this simply returns the character currently pointed to.

size_t tetraphilia::data_io::BufferedStream::PeekBytes ( const char *&  ptr,
size_t  minPeek,
const char *  arr,
int  mask 
)

Peek forward until a character C is encountered that satisfies both of the following criteria:

1) The bitfield at arr[C] masked with "mask" is nonzero, AND 2) C is at least minPeek bytes from the current position.

The start of the buffer will be returned in "ptr," and the number of bytes between ptr and C (or EOF, if no such character was encountered) is returned.

size_t tetraphilia::data_io::BufferedStream::PeekBytes ( const char *&  ptr,
size_t  numBytes 
)

Peek at the next numBytes characters in the stream. Guaranteed to retrieve the next numBytes unless EOF is encountered.

Returns the number of bytes available. This may be less than the number of requested bytes (if EOF is encountered), or more (if more bytes are available than were requested).
