29 // Generate text appearance for the given free-text annotation.
30 // If the appearance was generated successfully, write the appearance stream to the dictionary and return true.
31 // If the encoding for the contents is not supported, the AP is not written and false is returned.
32 // Clients can choose to write an image appearance in that case.
33 // 'lines' is the contents split into multiple lines and the width of characters for each line.
34 // 'line_height' is the height of line of text, descent is the font descent.
35 // 'wrap_lines' specifies whether or not individual lines are wrapped in /P marked content tags.
36 // 'support_cjk_text' specifies whether we should generate text appearance for CJK fonts, or 'fail' and fall-back to
37 // generating an image appearance. For Commenting calls, which supports image appearance-backed text annotations,
38 // we should specify 'no.' For Fill & Sign, which doesn't, continue using the potentially buggy text appearance route
39 // for now.
