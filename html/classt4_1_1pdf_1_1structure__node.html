Detailed Description

A union container for the node types in a structure tree.

The node types include the objects comprising a structure tree (as defined in the PDF 2.0 spec, ISO 32000-2:2020(E).

  • structure tree root (see section 14.7.2 "Structure hierarchy")
  • structure element (see section 14.7.2 "Structure hierarchy")
  • content item, also referred to as "structure content" (see section 14.7.5.1.1 "Content items")
  • object reference (see section 14.7.5.3 "PDF objects as content items")

◆ for_each_child()

void t4::pdf::structure_node::for_each_child ( const std::function< void(const structure_node &)>  callback) const

Iterates over each direct child of this structure_node and invokes a callback for each one.

Note that only structure tree root and structure element (types 'root' and 'element)' are considered to have "direct children."

This method can safely be called on all other types (including a default constructed instance), but nothing will happen. In particular, for structure content (type 'content'), the consituent mcid nodes are not enumerated by this method.

Parameters
callbackThe function to invoke for each child.

Returns the specified direct child for the node.

Note that only structure tree root and structure element (types 'root' and 'element)' are considered to have "direct children."

For supported types this throws an exception if the index is out of range.

This method can safely be called on other types, and returns a default-constructed structure_node regardless of the index. For default-cnostructed instances, however, calling this throws an exception.

◆ get_structure_type()

t4::string t4::pdf::structure_node::get_structure_type ( ) const

Returns a string representation of the node's structure type.

The value is obtained by calling get_structure_type() on the underlying node type. Note that some of these values are artificial, having no connection to any content in the PDF. Consequently this method is of little value. It is generally more meaningful and useful to call the type-specific get_structure_type() methods instead.

For reference, here are possible values as of this writing: structure tree root: "Root" structure element: the value of the element's /S key structure content: "PDEContainer" object reference: "Annot", "Form" or "OBJR" (for unsupported types)

Not valid to call on a default constructed instance (will throw an exception).

Returns
A string representing the node's structure's type.

◆ type()

structure_type t4::pdf::structure_node::type ( ) const

Returns an enumerated constant indicating the type of this structure_node.

Note that while there are specific 'structure_type' values for structure tree root, structure element and structure content, there is none for object reference. Instead, the type returned for an object reference is the type value for the referenced object. As of this writing, only link annotations and form elements are supported.

Returns
A structure_type representing the structure_node type.
