◆ permitted_operation

enum t4::pdf::permitted_operation
strong

Enumeration describing the permissions flags on an Store.

A permitted_operations bitfield is retrieved by calling Store::get_permissions(). This describes which operations the client should and should not be able to do given the permissions level derived from their password.

In the standard security handler implementation, if the client provides the owner password, the permissions level is necessarily permitted_operations_all. If the client provides the user password, the permissions level is specified by the value of the P key in the encryption dictionary (the value may be modified to accord with the dicta of the PDFRM, table 3.15.).

Note that the Store does not have any inherent ability to manage respect for these flags; it is the client application's responsibility to respect the permissions dictated by the security flags.

See PDFRM table 3.15 for a description of these various flags.
