28 /*
29  Create a data_store based on a model of the "fixed block length device" concept. This
30  class should be used when modeling a data_store with the following properties:
31 
32  1) Reads are expensive enough that storing subsets of the modeled bytes is worth
33  doing.
34  2) The underlying device can be read in a random-access manner, producing blocks of
35  fixed size (except for the last one in the file, of course)
36 
37  Suppose Device is a type that models "fixed block length device," and d is an instance
38  of Device:
39 
40  extern data_store_offset offset;
41  extern size_t sz;
42 
43  data_block db = get_block( d, offset, sz );
44 
45  The resulting data_block must represent the sz bytes at the specified offset for the
46  given device. A shorter block may be returned for the last block in the Device; this
47  will be presumed to be the end of the Device.
48 
49  The Device must not return different bytes given the same parameters. If this requirement
50  cannot be honored then an exception must be thrown.
51 
52  Implementers of get_block may presume that offset % block_granularity == 0, where
53  block_granularity was passed into make_fixed_length_caching_data_store.
54 
55  data_store_size sz = get_size( d );
56 
57  This is optional, but a call to get_size on d (using ADL) compiles then it should return
58  the length of the underlying store in bytes. While T4 will work very hard to request
59  a data_store's size, sometimes it must, and if this overload is not provided then it
60  will call get_block() repeatedly until it hits the end of the file--a potentially very
61  expensive operation. So if it's easy to provide get_size() (as it is for, say, files on
62  disk) then you should.
63 
64  Device must model MoveConstructible. Calls to get_block and get_size can be made from
65  multiple threads; it is expected that implementations will typically use a shared_ptr to
66  a mutable, stateful object whose methods are protected by a mutex.
67 
68  Given a model of fixed block length device d, create a data_store using make_fixed_block_size_caching_data_store:
69 
70  data_store ds = make_fixed_block_size_caching_data_store( std::move( d ), block_granularity );
71 */
72 template<typename Device>
73 data_store make_fixed_block_size_caching_data_store( Device d, size_t block_granularity = 4 * 1024 );
74 
75 /*
76  Create a data_store based on a model of the "variable block length device" concept. This
77  class should be used when modeling a data_store with the following properties:
78 
79  1) Reads are expensive enough that storing subsets of the modeled bytes is worth
80  doing.
81  2) The natural size of the blocks produced by the underlying data_store is unpredictable or
82  unknown.
83 
84  Suppose Device is a type that models "variable block length device," and d is an instance
85  of Device:
86 
87  extern data_store_offset offset;
88  extern size_t sz;
89 
90  std::pair<data_store_position, data_block> get_block( Device const& d,
91  data_store_offset const& offset, variable_block_size_block_stasher stash );
92 
93  The result is pair where result.second is a data_block containing the bytes stored by the
94  underlying device at offset result.first. This block must contain the byte requested by
95  the "offset" parameter, i.e., (result.first <= offset < result.first + result.second.get_size()),
96  unless offset is past the end of the data_store, in which case a zero-length block must be returned
97  and result.first must be offset.
98 
99  It is expected that the implementation of get_block may incidentally create unneeded data_blocks at
100  offsets before the one requested. These blocks can be passed to the "stash" functor; they will
101  be added to the cache with very low priority so that subsequent requests for those bytes are more
102  likely to result in cache hits:
103 
104  auto db = incidentally_created_block_at( offset );
105  stash( offset, std::move( db ) );
106 
107  The Device must not ever return conflicting results; for any two calls, if the returned blocks
108  overlap, they must report the same bytes. If this requirement cannot be honored then an exception
109  must be thrown.
110 
111  Device must model MoveConstructible. Calls to get_block can be made from multiple threads; it is
112  expected that implementations will typically use a shared_ptr to a mutable, stateful object whose
113  methods are protected by a mutex.
114 
115  Given a model of variable block length device d, create a data_store using
116  make_variable_block_size_caching_data_store:
117 
118  data_store ds = make_variable_block_size_caching_data_store( std::move( d ) );
119 */
120 template<typename Device>
121 data_store make_variable_block_size_caching_data_store( Device d );
122 
123 /*
124  Instances of this class are passed to a variable_block_size_caching_data_store's device's
125  get_block overload. Implementers may cache blocks that needed to be created using operator().
126 */
127 template<typename Device>
128 class variable_block_size_block_stasher
129 {
130  data_store_detail::variable_block_size_caching_data_store<Device> const& m_vbscdsb;
131 
132  friend class data_store_detail::variable_block_size_caching_data_store<Device>;
133 
134  variable_block_size_block_stasher( data_store_detail::variable_block_size_caching_data_store<Device> const& vbscdsb )
135  : m_vbscdsb( vbscdsb )
136  {}
137 
138 public:
139  /*
140  "Stash" a data_block in the cache that was created as a side effect of a request for a
141  block at a nonoverlapping offset. The provided block will be stored in the cache at low
142  priority.
143  */
144  void operator()( data_store_position const& offset, data_block const& block ) const;
145 };
