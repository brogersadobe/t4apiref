44 /*
45  A utility class used by different Annots to generate respective appearance streams. It can generate both kinds of appearances,
46  temporary (for rendering only) and persistent (changes are saved into annotation dictionary).
47  */
48 class appearance_generator
49 {
50 public:
51  /*
52  Usually indirect objects (such as streams) in t4 are referenced through the corresponding instances of t4::pdf::reference type.
53  When creating new ido through Store api, instead of an actual object, clients get back a reference which later can be resolved
54  to the object of concrete type. Unfortunately, temporary objects do not support notion of a reference by its nature, therefore
55  Store has to provide different api for operating temporary objects. This is problematic for cases like appearance_generator,
56  where we could have the same implementation for generating temporary and regular appearances, but have to deal with differences
57  in ido handling.
58  The main purpose of 'ido_handler' class is to provide unified api to idos operating functionality required by appearance_genearator.
59  When creating new instance of appearance genearator, client can specify what kind of appearance stream it will generate (temporary vs persistent)
60  by passing corresponding 'is_temporary_appearance' flag into constructor. When appearance generator require to create new indirect object,
61  it has to do so through the 'm_ido_handler' instance, which will create ido of proper type based on 'is_temporary_appearance' flag.
62 
63  Future maintainers: when making changes in 'appearance_generator', always use 'ido_handler' for all idos manipulations. Feel free
64  to extend 'ido_handler' with new APIs if it's required for your needs.
65  */
