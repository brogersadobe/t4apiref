28 /*
29 All of applications want to use thin stroke when possible so long as they
30 fall within desired speed and imaging_model fidelity constraints;
31 This is because thin strokes are not intended to be a performance boost
32 (though they often are) but rather a more attractive stroke.
33 As such it is highly desirable to use thin stroke so long as it is
34 deemed "legal". I think the three reasonable levels of thin stroking legality
35 are:
36  ThinStrokeNever
37  Some algorithms require "stroke evelope outlines"
38 
39  ThinStrokeAggressive
40  This is what Illustrator wants during patterns: aggressively go thin
41  for performance in patterns. For now it would be used when strokeadjust
42  is false and oldThinStrokePolicy is true.
43 
44 
45  ThinStrokeHighlyCompliant
46  This is the default policy, used everywhere except for the above
47  case. This is what almost all of the applications wants. Acrobat may claim to want fully compliant but they do not have that today and we are skeptical that they would like it if they got it.
48 
49 
50  ThinStrokeFullyCompliant
51  This is used by the flattener where a thin stroke is unusable.
52  Arguably Acrobat might want this, but I doubt it.
53 
54 
55 Here is the threshold table
56 
57  Aggressive HighlyCompliant FullyCompliant
58  SA false: w < 1.000 w < 0.500 w == 0
59  SA true: w < 1.5 w < 1.185 w < 0.5 // see strokeadjust section of PSRedBook
60 */
