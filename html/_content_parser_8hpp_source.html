561 /*
562  There exist PDF content streams (see DCMCD-431) with constructs like this:
563 
564  [ (Hello ) 10.0 Tc (World!) ] TJ
565 
566  This is invalid PDF; Tc operators are not allowed inside TJ operators. Nevertheless
567  desktop accepts this and even applies the operator to the text state (interestingly,
568  to the entire text array at TJ time--in the above example the character spacing is
569  applied to every character in the string, not just the (World!) part).
570 
571  We tolerate this by specializing ArrayParser for ContentStreamObjTraits and effectively
572  forwarding calls to the Tc operator to a "parent" content parser. For now we only
573  handle the Tc operator because that's the only such operator we've seen in the wild,
574  and we'd rather not contribute to the proliferation of malformed PDF by tolerating
575  bad constructs that no one is producing yet.
576 
577  If in the future we need to expand the space of operators tolerated during parsing of
578  arrays in contents streams, add the new operator to TokenDispatch_Generator.cpp, rebuild
579  the parser tables, and modify this class to forward the call to the parser client.
580 */
581 #ifdef TETRAPHILIA_PDF_PERMISSIVE
582 template<>
583 class ArrayParser<content::ContentStreamObjTraits<t4::app_traits>> : public ObjectStoreParser<content::ContentStreamObjTraits<t4::app_traits>>

586  using object_traits = content::ContentStreamObjTraits<app_traits>;
