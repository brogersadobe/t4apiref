38 // Generate text appearance for the given free-text annotation.
39 // If the appearance was generated successfully, write the appearance stream to the dictionary and return true.
40 // If the encoding for the contents is not supported, the AP is not written and false is returned.
41 // Clients can choose to write an image appearance in that case.
42 // 'lines' is the contents split into multiple lines and the width of characters for each line.
43 // 'line_height' is the height of line of text, descent is the font descent.
44 // 'wrap_lines' specifies whether or not individual lines are wrapped in /P marked content tags.
45 // 'support_cjk_text' specifies whether we should generate text appearance for CJK fonts, or 'fail' and fall-back to
46 // generating an image appearance. For Commenting calls, which supports image appearance-backed text annotations,
47 // we should specify 'no.' For Fill & Sign, which doesn't, continue using the potentially buggy text appearance route
48 // for now.
49 bool generate_text_appearance_for_free_text_annotation(annotation& annotation, t4::vector<std::pair<t4::string, t4::vector<t4::real>>> const& lines, t4::real line_height, t4::real descent, t4::pdf::font_info const& font_info, wrap_lines_in_p_marked_content const& wrap_lines = wrap_lines_in_p_marked_content::no, support_cjk_text const& support_cjk = support_cjk_text::yes);
