207  /* The StartTransaction Variants:
208  * Take the following four kinds of parameters -
209  * (a) TransactionType - Allowed values are STORE_READ (for transactions that guarantee not to write to tetraphilia's store) and STORE_WRITE
210  * (b) Functors - Four categories of functors which serve as a callback for pre-undo, post-undo, pre-redo and post-redo operations
211  * (c) Transaction String Pair - An std::pair of std::strings that provide a user displayable message for undo/redo
212  * (d) Client Data - A optional client managed memory pointer
213  *
214  * All functors are executed on BG thread. The client code must not execute any store modification operations in any of the functors.
215  * The pre undo/redo functors are meant for reading/querying the state of the system before undo/redo.
216  * Normally, clients should invoke asynchronous UI operations (like PostToUIThread) in post undo/redo functors.
217  *
218  * All "StartTransaction" variants return the transaction number started.
219  */
