Detailed Description

Simple streaming interface for writing data.

Implementations may not be threadsafe. All methods must be called on the same thread.

size_t t4::pdf::write_stream::bytes_written ( ) const
inline

Returns the number of bytes successfully written to the stream.

Will match the cumulative number of bytes passed to write() as long as no writes failed.

The value is not affected by calls to flush(). In other words the count includes any bytes that the implementation may have buffered internally.

May be called after either set_error() or close() has been called.

Returns
The number of bytes successfully written to the stream.

◆ close()

void t4::pdf::write_stream::close ( )
inline

Closes the write_stream after flushing any remaining buffered bytes.

Also performs implementation-specific clean up. Implementations may call this in their destructor, but callers should assume this and so should always call this directly when done with the stream.

Any further attempts to write to the stream will fail. Does nothing (and is benign) if called multiple times.

Will never throw. Suppresses thread_interrupted and eats all other exceptions thrown during execution. Safe for use in a RAII destructor.

◆ set_error()

void t4::pdf::write_stream::set_error ( t4::exception_ptr const &  ex)
inline

Notifies the stream that an error has occurred.

Any further calls to write() or flush() will fail (with an implementation-defined exception). There may be other implementation-specific handling, such as tossing any data accumulated so far, invalidating the container the stream is writing into, triggering error handlers in listeners, etc.

Called internally by write() and flush() if either of these methods throws an exception. Clients may also call directly to invalidate the stream in case of external errors.

Does nothing (and is benign) if called multiple times, or if called after close().

Will never throw. Suppresses thread_interrupted and eats all other exceptions thrown during execution.
