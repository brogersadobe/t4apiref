Detailed Description

Class representation of a font dictionary.

Can only be used to represent a "real" font. This does not include CIDFontType0 or CIDFontType2, which can only be used in a Type0 font's DescendantFonts.

enum t4::pdf::font::subtype
strong

Possible values of Subtype in a font dictionary.

Only includes "real" font types, e.g. for font dictionaries that can appear in a Resources dictionary, a class map entry, etc. Does not include CIDFontType0 or CIDFontType2 which can only be used as a DescendantFonts value for a Type0 font. From the PDF Reference: "a CIDFont is not actually a font."

◆ get_postscript_name()

optional<t4::string> t4::pdf::font::get_postscript_name ( ) const

Returns the PostScript name of the font, or none if there is no PostScript name.

The PostScript name is based on the BaseFont value, which is required for all subtypes except Type3 where it is optional. So this method can only return none for Type3 fonts.

Returns
The font's PostScript name, or none if it doesn't have one.

◆ get_program_type()

optional<program_type> t4::pdf::font::get_program_type ( ) const

Returns the format of the embedded font, or none if there is no embedded font.

Handles exceptions internally (such as for errors in the FontDescriptor) and treats these as if there is no embedded font, returning 'none'.

Non-error cases in which this may return 'none' are Type 3 fonts (where a FontDescriptor is optional but rare) and for the standard 14 PostScript fonts (prior to PDF 2.0).
